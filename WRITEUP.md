## The challenge

Write a market-making strategy for a simulated binary prediction market order book. An informed trader punishes bad quotes. Uninformed retail traders are your profit source. A static competitor sits on the book. 200 simulations per eval, ranked by mean edge.

That's all you need to know. The rest is about how the strategy was found.

## Autoresearch with parallel agents

Inspired by [Karpathy's autoresearch](https://github.com/karpathy/autoresearch). Claude Code as a team of parallel research agents, each exploring a different direction at the same time.

```
         ┌──────────────────┐
         │  Pick directions  │
         │  Spawn agents     │
         └────────┬─────────┘
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
 ┌────────┐  ┌────────┐  ┌────────┐
 │Agent 1 │  │Agent 2 │  │ ... 20 │
 │Modify  │  │Modify  │  │Modify  │
 │Eval    │  │Eval    │  │Eval    │
 │Report  │  │Report  │  │Report  │
 └───┬────┘  └───┬────┘  └───┬────┘
     └────────────┼────────────┘
                  ▼
         ┌──────────────────┐
         │  Keep winners     │
         │  Update best      │
         │  Repeat           │
         └──────────────────┘
```

At peak, 20 agents running at once. Each gets a specific hypothesis, the current best strategy, and a target to beat. They modify code, run evals, report results. I never wrote strategy code. I never wrote eval scripts. The agents wrote 47 automated sweep scripts on their own — grid searches, genetic algorithms, parameter sweeps.

When Claude Code agents got stuck, I occasionally ran Codex agents on the same challenge — different model, different architectural ideas. Fed their techniques back into the Claude Code agents for cross-pollination.

**+$14.61 to +$25.39.** Every existing strategy stopped trading when the competitor's book emptied out. But that's when retail just consumed the competitor's orders — the single best moment to trade. **+$25 to +$44.** The breakthrough. Covered in the next section.

## Starting from scratch to escape a local optimum

This was the single most important thing that happened.

We had 700+ strategies. The agents had been iterating for hours. Every modification was a small delta on the same architecture. Progress had flatlined around +$25.

I wrote this prompt:

```
You are trying to win a prediction market making
challenge. The current best strategy scores +25 avg.
You should try a COMPLETELY DIFFERENT approach
from scratch.

Key insights that work: skip cs=1, 5% floor exploit,
None-handling at tick 1/99. But the SIZING is wrong
— try decomposing into retail capture vs arb
avoidance.

Write to strategy_2020.py. Evaluate on seeds
0, 500, 1000, 2000. Target: beat +25 avg.
Iterate up to 3 times.
```

This agent had zero access to our prior strategies. It decomposed the problem differently — retail capture and arb avoidance as separate concerns, modeled with a Gaussian probability function:

```
size = k * retail_mult * (1 - arb_prob * damping)
```

Where `arb_prob = exp(-spread^2 / (2 * ev^2))` models the probability of getting picked off per side. This was architecturally different from anything the previous 700 strategies had tried.

+$25 to +$44. One agent. One fresh start. More gain than all previous work combined.

The agents that knew the most about our prior work were the least likely to find this. When progress plateaus, resetting beats iterating. This is the autoresearch equivalent of escaping a local optimum, and it's the single most transferable lesson from this project.

## What didn't work

The agents killed ideas as efficiently as they found them. Across 1,000+ experiments, a few examples:

Retail quantity capping looked promising in theory — cap order size to reduce arb exposure. An agent tested 5 multipliers across 4 seeds:

```
Cap    Retail lost   Arb saved   Net
2.0x    -$17.59       +$7.97    -$9.62
3.0x    -$14.79       +$6.91    -$7.87
5.0x    -$10.84       +$5.30    -$5.54
8.0x     -$6.97       +$3.80    -$3.17
```

Retail loss was 2x the arb savings at every multiplier. Retail arrives in bursts, so any cap loses more than it saves. Dead in 10 minutes with data.

Other confirmed dead ends:
- Multi-level quoting (-$3.46): gives the arbitrageur more to eat
- Bayesian mid estimation (-$3): hand-tuned heuristics beat the principled approach
- Aggressive wide-spread sizing (-$1.64): more size = proportionally more arb exposure

After 100+ sweep agents — each one systematically testing hundreds of parameter combinations across multiple seeds — the parameter landscape around the final strategy was confirmed flat. Nothing left to find.

## What I actually did for 8 hours

My job for 8 hours:

- Read agent results
- Pick which directions to explore next
- Say "keep going" or "try something different"
- Occasionally tell an agent to throw everything away and start fresh

The agents wrote all the code, all the eval scripts, all the infrastructure. They built their own sweep scripts, genetic algorithms, and multi-seed evaluation harnesses. The winning strategy has 14 subsystems and ~900 lines of Python I couldn't have written.

But as Karpathy [pointed out](https://github.com/karpathy/autoresearch), the human is still in the loop — monitoring, steering, deciding when to reset. For 8 hours I was the slowest part of the system. The agents could have run faster without me.

The next layer of abstraction is an agent that does what I did: watch results, pick directions, occasionally say "start over." When that exists, the human isn't the bottleneck anymore. I'm looking forward to it.
